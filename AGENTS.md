# AGENTS.md

1. Qualquer alteracao que produza artefato deve ser mergeada via PR na branch `develop`. Nao misture alteracoes de escopos diferentes no mesmo commit ou PR.
2. Utilize Gitflow e commits semanticos sempre.
3. Toda nova mudanca deve ser iniciada criando uma branch apropriada a partir da branch base correta.
4. Toda implementacao guiada pelos prompts deve obedecer a issue correspondente em `docs/github-issues-agendaedu.md`, usar a branch sugerida nessa issue ou justificar divergencia no PR, e vincular a issue ao Pull Request.
5. Toda solicitação deve ser respondida criando um `.md` na pasta `docs/`, não via CLI.
